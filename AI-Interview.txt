Build me a full-stack AI-Powered System Design Interviewer web application. Here are the complete requirements:

 **PROJECT OVERVIEW**
A web app that simulates a real Amazon/Google-style system design interview. The user picks a system to design, an AI acts as their interviewer asking follow-up questions, a live architecture diagram builds on screen as they talk, and at the end the AI scores them with detailed feedback.

 **TECH STACK**
 - Frontend: React + Tailwind CSS
 - AI: Claude API (claude-sonnet-4-20250514) using streaming responses
 - Diagrams: React Flow for real-time architecture diagrams
 - Database: AWS DynamoDB to store interview sessions and scores
 - Deployment: Vercel
 - CI/CD: GitHub Actions

 **PAGES & FEATURES**

Page 1 — Landing Page:
> - Clean hero section with title "AI System Design Interviewer"
> - Subtitle: "Practice Amazon & Google-style system design interviews with AI"
> - Grid of preset system cards to choose from: Design Twitter, Design Netflix, Design Uber, Design WhatsApp, Design URL Shortener, Design YouTube, Design Amazon, Design a Chat App
> - A text input to type a custom system to design
> - A "Start Interview" button
> - Show a "Recent Sessions" section at the bottom pulling from DynamoDB
>
> Page 2 — Interview Page (split screen layout):
> - LEFT SIDE: Chat interface (like WhatsApp/ChatGPT)
>   - Shows the conversation between user and AI interviewer
>   - AI messages have an Amazon-style interviewer avatar
>   - Text input at bottom to type responses
>   - "End Interview" button
>   - Interview timer showing elapsed time
> - RIGHT SIDE: Live Architecture Diagram
>   - Built with React Flow
>   - Starts empty
>   - As the user mentions components (database, cache, load balancer, API, CDN, queue, microservice), automatically add the corresponding node to the diagram
>   - Use icons for each component type
>   - Nodes should be draggable and repositionable
>   - Draw edges/arrows between related components automatically
>   - Components to detect and render: Client, Load Balancer, API Gateway, Web Server, Database (SQL), Database (NoSQL), Cache (Redis), Message Queue, CDN, Storage (S3), Microservice, Authentication Service
>
> Page 3 — Results Page:
> - Overall score out of 100 with a circular progress indicator
> - Breakdown scores for: Requirements Gathering, Scalability, Database Design, API Design, Trade-off Discussion
> - "What you did well" section in green
> - "What you missed" section in red
> - "Model Answer" section showing what an ideal answer looks like
> - "Try Again" and "New System" buttons
> - Save this session to DynamoDB with timestamp, system designed, score, and summary
>
> **AI INTERVIEWER BEHAVIOUR**
> The Claude API system prompt should be:
> "You are a senior Amazon software engineer conducting a system design interview. You are professional, encouraging but challenging. Start by asking about functional and non-functional requirements. Then probe into: scalability, database choices, API design, caching strategy, handling failures, and trade-offs. Ask one focused question at a time. After 8-12 exchanges, end the interview by saying INTERVIEW_COMPLETE followed by a JSON scoring object with fields: overallScore (0-100), requirementsScore (0-100), scalabilityScore (0-100), databaseScore (0-100), apiScore (0-100), tradeoffScore (0-100), strengths (array of strings), improvements (array of strings), modelAnswer (string). Also throughout the conversation, whenever the user mentions a technical component, output a line starting with COMPONENT: followed by the component name so the frontend can update the diagram."
>
> **AWS DYNAMODB SETUP**
> - Create a table called "interview-sessions"
> - Fields: sessionId (partition key), timestamp, systemDesigned, overallScore, duration, summary
> - Use AWS SDK v3 (@aws-sdk/client-dynamodb)
> - Store a new session every time an interview is completed
> - Fetch recent sessions on the landing page
> - Use environment variables for AWS credentials: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_REGION
>
> **GITHUB ACTIONS CI/CD PIPELINE**

> - Uses GitHub Secrets for: VERCEL_TOKEN, VERCEL_ORG_ID, VERCEL_PROJECT_ID, ANTHROPIC_API_KEY, AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY
